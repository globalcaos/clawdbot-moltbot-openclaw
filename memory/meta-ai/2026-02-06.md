### 1. YOLOv26 ‚Äî Edge-First Object Detection (BREAKTHROUGH)

**Source:** [Ultralytics / LearnOpenCV](https://learnopencv.com/yolov26-real-time-deployment/)

**What:** YOLOv26 released January 2026 with radical simplification:
- **NMS-free detection** ‚Äî One-to-One head eliminates post-processing
- **No DFL** ‚Äî Removed Distribution Focal Loss for clean INT8/FP16 quantization
- **MuSGD optimizer** ‚Äî LLM-inspired training from Muon optimizer
- **STAL + ProgLoss** ‚Äî Small target awareness built-in

**Benchmarks:**
| Model | mAP (50-95) | CPU (ONNX) | GPU (T4 TensorRT) |
|-------|-------------|------------|-------------------|
| YOLOv26-n | 40.9 | ~39ms | 1.7ms |
| YOLOv26-m | 53.1 | ~220ms | 4.7ms |

- **43% faster on CPU** vs YOLOv11/12
- Sub-2ms latency on T4 for nano model

**YOLOE-26:** Open-vocabulary variant ‚Äî set classes at runtime via `model.set_classes(["hard hat", "safety vest"])`

**Source:** [DEV.to ‚Äî Daydream](https://dev.to/viborc/real-time-open-source-ai-video-generation-is-here-and-heres-what-you-can-build-with-it-41d5)

**Key innovation:** Pipeline architecture ‚Äî chain depth estimation (Depth Anything V2), pose detection, segmentation (SAM2), then diffusion, then upscaling (RIFE/FlashVSR). All real-time.

### 3. 3D Gaussian Ray Tracing for Tomographic Reconstruction

**Source:** [arXiv:2602.01057](https://arxiv.org/abs/2602.01057)

**What:** New approach to 3D reconstruction extending 3D Gaussian Splatting:
- Analytically computes line integrals through 3D Gaussians (vs affine approximation)
- Explicit ray origin/direction control for geometric corrections
- More physically consistent forward projection model
- Applicable to CT and PET imaging

**Why interesting:** The ray-tracing formulation could improve pallet/object 3D reconstruction from multi-view camera setups.
